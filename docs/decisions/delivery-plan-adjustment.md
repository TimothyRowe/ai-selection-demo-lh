# 交付计划调整记录 - 2026-06-20

## 核心变更：一期二期合并交付

采用 vibe coding 模式，选产品 + 竞品洞察一起上线。

---

## 改动清单

### 1. 侵权词筛选 + 以图搜图查重 → 移至选产品结果列表内手动触发

- **原方案**：L2 异步精筛自动执行
- **新方案**：在选产品结果列表工具栏提供手动触发按钮，勾选后批量执行
- **原因**：提高筛选产品的响应效率，舍弃一步到位过滤的体验
- **影响**：结果池不再有"待精筛→精筛中"的自动状态流转

### 2. 隐藏"合规风险"和"认证需求"字段

- **原方案**：结果列表中展示风险标签和认证标签
- **新方案**：暂时隐藏这两个字段，不在结果列表中展示
- **原因**：暂时没有内外部数据或确认的第三方知产审核数据支撑
- **影响**：demo 中 risk/riskDetail/certs/certsOwned 字段暂不实现

### 3. 删除选产品模块顶部统计数据

- **原方案**：顶部展示汇总统计（总数/均价/平均评分等）
- **新方案**：删除，不展示
- **原因**：无用信息，干扰视觉

### 4. 竞品异动合并入竞品洞察模块

- **原方案**：选产品模块中有竞品异动内容，竞品洞察是独立模块
- **新方案**：合并为一个模块"竞品洞察"，异动作为第一个 Tab
- **影响**：选产品模块专注筛选，竞品相关分析统一归入竞品洞察

### 5. 竞品洞察模块位置调整

- **原方案**：竞品洞察与选产品并列
- **新方案**：竞品洞察挪至选品结果池模块下方
- **逻辑**：用户流程为 筛选→入池→对池中商品做竞品分析，位置跟随使用时序

### 6. 选品结果池去除评估流程

- **原方案**：含上级评估状态（待评估/已评估通过）
- **新方案**：去除评估流程，专员自主推送 ERP
- **状态**：待推送 → 推送成功/推送失败/作废
- **原因**：业务团队无上级评估环节

### 7. 竞品异动响应动作调整

- **原方案**：运营工单 / 选品工单按钮
- **新方案**：
  - 路径A → "建议通知运营"（v1.0 无 ERP 工单对接，仅文字提示）
  - 路径B → "加入结果池"（一键入池）
- **原因**：v1.0 未接入 ERP 运营工单机制

### 8. 竞品洞察页面改为 Tab 切换结构

- **原方案**：折叠卡片 + 监控卡片网格 + 点击展开详情
- **新方案**：4 个 Tab 平铺（异动机会 / BSR榜单监控 / 刊登链接监控 / 监控管理）
- **原因**：原交互层级混乱，监控卡片与异动结果关系不清晰

### 9. BSR/刊登 Tab 增加公共统计 + 按监控人区分

- **新增**：顶部监控选择器（按创建人区分不同监控任务）
- **BSR 公共统计**：稳定在榜、新品进入、新品进入率、卖家国籍占比、搜索供需比、有品牌占比
- **刊登公共统计**：监控ASIN数、排名异动、差评告警、状态正常
- **操作列**：评论洞察 + 趋势洞察（替代原来的"洞察报告"/"评论"按钮）

### 10. 监控管理改为表格形式

- **原方案**：大卡片网格布局
- **新方案**：紧凑表格（名称/类型/范围/频率/归属/异动数/状态/操作）
- **约束**：每人最多 5 个监控任务

---

## 调整后的模块结构（v1.0 最终版）

```
1. 选产品（筛选漏斗）
   - 多维度条件筛选 + 筛选方案保存
   - 结果列表 + 工具栏（手动触发：侵权词检测、以图搜图查重）

2. 选品结果池
   - 多维度展示 + 列选择器（popover）
   - 推送状态（待推送/成功/失败/作废）+ 推送日志
   - 统计：已推送 / 已作废 / 推送并开品完成

3. 竞品洞察（Tab 切换）
   ├── 异动机会（路径A建议通知运营 / 路径B加入结果池）
   ├── BSR 榜单监控（按监控人切换 + 公共统计 + ASIN明细）
   ├── 刊登链接监控（按监控人切换 + 公共统计 + ASIN明细）
   └── 监控管理（表格形式，每人限5个）
```
