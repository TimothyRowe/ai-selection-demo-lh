# AI 智能选品系统 — 立项方案（更新于 2026-06-22）

## Context

某跨境电商企业主营家居/工业科学/汽配五金，海外仓重仓模式（多国仓）。当前选品依赖人工在卖家精灵 SaaS 上手动筛选，效率低且决策质量不稳定。本项目目标：构建 AI 辅助选品系统，逐步从「人工筛选」走向「机器推荐+人工复核」，降低选品失误率、提升开品效率。

优先服务业务产品开发团队（新组建），目标借助 AI 辅助提升选品效率。

立项时间：2026-06-15 | v1.0 目标上线：2026-09-30

### 技术约束

| 项目 | 现状 |
|------|------|
| 交付形式 | 独立系统，通过 API 推送 ERP 产品库 |
| 技术栈 | Java（Spring Boot 3.x），独立 vibe coding 模式 |
| 选品系统团队 | 架构师 + 后端开发 |
| ERP 团队 | Java 后端，负责 ERP 产品库日常研发，8 月可介入接口对接 |
| ERP 对接 | 需 ERP 团队开发「创建开品单」接口 |
| 卖家精灵 API | 商务洽谈中，选产品实时 API + ProBoost 类目快照分开购买 |
| 内部数据 | 侵权词库（已有）、产品库图片用于以图搜图（已有） |

## v1.0：选品闭环 + 竞品洞察（6.15 — 9.30）

### 时间分配

| 阶段 | 时间 | 人员 | 产出 |
|------|------|------|------|
| 产品设计 + 技术方案 | 6.15 - 6.30（2周） | 产品经理 | PRD、Demo原型、API 文档整理、技术方案评审 |
| 开发阶段 | 7.1 - 9.15（11周） | 架构师 + 后端 | 系统开发（vibe coding） |
| ERP 联调 | 8.1 - 9.15（与开发并行） | ERP 团队 | 推送接口开发 + 联调 |
| 测试 + 试运行 | 9.15 - 9.30（2周） | 全员 + 业务团队 | 内部验收、选品组试用 |

### 目标

跑通「筛选 → 手动风控 → 结果池 → 推送 ERP」+ 「竞品异动监控 → 快速响应」双链路。

### 功能模块

| 模块 | 说明 | 核心能力 |
|------|------|----------|
| 选产品 | 对接卖家精灵实时 API | 多维度筛选（站点/类目/价格/BSR/销量/评分/增长率/尺寸/配送）、筛选方案保存（私有/团队/系统） |
| 手动风控 | 用户触发，非自动 | 结果列表工具栏勾选执行：侵权词库匹配 + 以图搜图去重（与 ERP 已有 SKU 比对） |
| 选品结果池 | 汇集筛选结果 | 多维度字段展示（列选择器）、推送状态管理（待推送/成功/失败/作废）、推送日志 |
| 推送 ERP | 打通 ERP 产品库 | 选定商品 → 推送 ERP → 自动填充可映射字段 → 开发专员补全资料 → 开品 |
| 竞品洞察 | Tab 切换结构 | 异动机会（路径A/B响应）、BSR榜单监控、刊登链接监控、监控管理（每人限5个） |

### v1.0 不含

- 运营工单对接 ERP（仅文字提示"建议通知运营"）
- 合规风险/认证需求字段（无数据源）
- AI 评分报告打分算法（待梳理）
- 1688 自动寻源
- 自动开品
- 上级评估流程（专员自主推送）

## v1.1：竞品洞察深化 + ERP 闭环（10.15 — 11.30）

### 目标

补齐 v1.0 中预留的能力缺口，实现运营工单自动创建 + 评论洞察闭环。

### 功能模块

| 模块 | 说明 |
|------|------|
| 运营工单对接 | 异动机会路径A → 自动创建 ERP 运营工单 |
| 评论情绪洞察 | ProBoost 评论数据 → AI 提取痛点 → 差异化建议 |
| AI 评分报告 | 打分算法落地，多维度加权评分 |
| 利润计算 | FBA 费用 API 对接，精细化利润估算 |
| 合规风险 | 第三方知产审核数据源对接后恢复 |

### 前置条件

- v1.0 内测验证通过
- ProBoost 数据采购落地
- ERP ECN 工单接口开放
- 第三方知产/合规数据源确认

## v2.0：供应链 + 多类目扩展（次年 1 月+）

### 目标

对接 1688 实现货源寻源，扩展多类目支持。

### 核心能力

| 能力 | 说明 |
|------|------|
| 1688 API 对接 | 以图搜图 API 回流 + 供应商匹配 + 关键词搜品 |
| 多类目扩展 | 规则模板 + 类目差异化权重配置 |
| 卖家监控 | 竞争对手店铺动态（上新/下架/价格/广告位） |
| ERP 深度打通 | 选品结果自动创建开品流程 |

## v3.0：智能化 + 自动化（持续迭代）

### 核心能力

| 能力 | 说明 |
|------|------|
| AI 自动选品 | 基于历史数据训练，自动跑分推荐 → 结果池 |

| 全自动流程 | 高置信度选品结果直通 ERP，减少人工节点 |
