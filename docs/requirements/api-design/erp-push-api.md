# ERP 对接接口规范

> 选品系统与 ERP 产品库的接口对接方案。ERP 团队 8 月介入开发，7 月需完成字段规范对齐。

## 对接架构

```
选品系统                          ERP 产品库
┌──────────┐    创建开品单     ┌──────────┐
│ 结果池   │ ──────────────→  │ 产品库   │
│          │                   │          │
│ 推送状态 │ ←──────────────  │ 回调通知 │
│ 更新     │   推送结果回调    │          │
└──────────┘                   └──────────┘
```

## 通信方式

- 协议：REST API（HTTPS）
- 认证：待定（建议 internal token / 签名机制）
- 数据格式：JSON

## 接口清单

### ERP 侧需提供（ERP 团队开发）

| # | 接口 | 方向 | 优先级 | 说明 |
|---|------|------|--------|------|
| 1 | 创建开品单 | 选品→ERP | P0 | 核心接口，推送选品结果创建开品单 |

| # | 接口 | 方向 | 优先级 | 说明 |
|---|------|------|--------|------|
| 1 | 推送结果回调接收 | ERP→选品 | P0 | 接收 ERP 回调，更新推送状态 |

## 接口详情

### 1. 创建开品单（选品→ERP）

**POST** `/api/erp/product/create`

#### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| requestId | String | 是 | 选品系统生成的唯一请求ID，用于幂等和回调关联 |
| source | String | 是 | 固定值 "AI_SELECTION"，标识来源 |
| operator | String | 是 | 推送人（选品系统用户ID） |
| productData | Object | 是 | 产品数据，见下方字段映射表 |
| └ asin | String | 是 | Amazon ASIN |
| └ marketplace | String | 是 | 站点（US/UK/DE/FR/IT/ES/AU/CA/JP） |
| └ title | String | 是 | 商品标题 |
| └ imageUrl | String | 否 | 主图 URL |
| └ brand | String | 否 | 品牌 |
| └ categoryPath | String | 否 | 类目路径 |
| └ price | BigDecimal | 否 | 售价 |
| └ bsr | Integer | 否 | BSR 排名 |
| └ rating | Float | 否 | 评分 |
| └ ratings | Integer | 否 | 评分数 |
| └ monthlyUnits | Integer | 否 | 月销量 |
| └ fulfillment | String | 否 | 配送方式（FBA/FBM/AMZ） |
| └ sellerNation | String | 否 | 卖家国籍 |
| └ availableDate | String | 否 | 上架时间 |
| └ ... | | | **其余字段由你整理补充** |
| callbackUrl | String | 是 | 回调地址 |

#### 响应

```json
{
  "code": "OK",
  "message": "受理成功",
  "data": {
    "erpOrderId": "ERP-2026-001234"
  }
}
```

> 注意：响应仅表示 ERP 受理请求，最终结果通过回调通知。

---

### 2. 推送结果回调（ERP→选品）

**POST** `{callbackUrl}`（选品系统在创建时传入）

#### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| requestId | String | 是 | 对应创建时的 requestId |
| erpOrderId | String | 是 | ERP 侧开品单号 |
| status | String | 是 | SUCCESS / FAILED |
| failReason | String | 否 | 失败原因（status=FAILED 时必填） |
| createdAt | String | 是 | 开品单创建时间（ISO 8601） |

#### 选品系统处理逻辑

| 回调 status | 选品系统动作 |
|-------------|-------------|
| SUCCESS | 推送状态更新为「推送成功」，记录 erpOrderId |
| FAILED | 推送状态更新为「推送失败」，记录失败原因，支持重新推送 |

---

### 3. 查询开品单状态（选品→ERP，兜底）

**GET** `/api/erp/product/status?requestId={requestId}`

用于回调丢失或超时未收到回调时主动查询，逻辑同回调字段。
