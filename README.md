# 数字化选品系统 - 交互原型Demo

## 项目概述

跨境电商AI辅助选品系统的交互原型，用于验证产品设计逻辑和演示系统能力。纯前端HTML实现，无需后端服务。

## 访问方式

直接浏览器打开 `selection-demo.html` 即可体验完整系统。

## 系统模块（5个页面 + 3个弹窗）

### 页面

| 模块 | 功能 | 数据源 |
|------|------|--------|
| 选产品 | 6行筛选面板 + 产品表格 + 合规风险 + 认证标识 | 卖家精灵API |
| 选品结果池 | AI筛选/人工筛选/竞品异动 三来源汇总 + 状态流转 | 系统内部 |
| AI自动选品 | 5维评分引擎 + 硬性过滤 + 内部数据修正 + 竞品异动捕捉 | 卖家精灵+ERP |
| 货源寻源 | 1688以图搜图 + 供应商匹配 + 自动询盘 | 1688 API |
| 竞品洞察 | BSR异动监控 + 评论挖掘 + AI开品建议 | Proboost+Amazon |

### 弹窗

| 弹窗 | 触发位置 | 功能 |
|------|----------|------|
| 产品详情分析 | 选产品表格点击 | 6 Tab: 概览/BSR趋势/关键词/竞品/利润/AI报告 |
| 竞品异动监控 | 选产品统计卡 | 17个异动机会列表(断货/骤降/下架) |
| 洞察报告 | 竞品洞察表格"洞察报告"按钮 | 评论原文+痛点聚类+AI评分+行动步骤 |
| 新建监控 | 竞品洞察"新建监控"按钮 | 选类型→填参数→设触发条件 |

## 竞品洞察模块详细设计

### 监控类型（3+1种）

- **BSR榜单监控**：按类目监控Top N，检测进出榜/排名异动/价格变动
- **刊登链接监控**：指定ASIN，盯单品BSR/价格/库存/评论变化
- **卖家监控**（V2规划）：监控指定店铺的上新/下架/价格/广告位变化
- 支持多角色创建：系统管理员(全局) / 团队(组长) / 个人(专员)

### 异常类型标签体系

| 标签 | 含义 | 颜色 |
|------|------|------|
| 爆品信号 | 排名飙升50+位 | 红 |
| 快速攀升 | 排名上升20+位 | 红 |
| 低评高升 | 评分低但排名升=需求旺产品差=机会 | 黄 |
| 低价冲榜 | 价格低于均价进入Top5 | 红 |
| 断崖下跌 | 排名大幅下跌=断货/差评 | 蓝 |
| 持续衰退 | 持续下跌=供应链/质量问题 | 蓝 |
| 新品入榜 | 新进Top100 | 绿 |
| 新品低评 | 新进且评分低=可改进开品 | 绿 |

### AI开品建议算法（5维评分）

| 维度 | 权重 | 数据来源 |
|------|------|----------|
| 痛点严重度 | 30% | 评论负面率×频次 |
| 供应链可解 | 25% | 供应商定制能力匹配 |
| 市场空间 | 20% | 需求量×客单价 |
| 竞争壁垒 | 15% | 技术门槛+差异化 |
| 内部优势 | 10% | 在售类目经验+动销 |

### 洞察报告弹窗结构

1. 产品信息 + 异常标签 + Amazon跳转
2. 评论原文（英文+中文翻译，按星级筛选）
3. AI痛点聚类（占比标签排序）
4. AI开品建议（5维评分+综合分+差异化方案）
5. 行动步骤（6步时间线+窗口期判断）

## 方法论来源

基于行业私董会大卖家分享的AI赋能方法论：
- **Fogg行为模型**：行为 = 动机 × 能力 × 提示
- **双数据思维**：大数据找共性(BSR/销量) + 厚数据找特点(评论/售后)
- **反常识选品顺序**：先建痛点框架 → 再选市场 → 最后评论验证

## 实际数据验证

以美国站 Cushions（坐垫）类目为例：
- 数据源：Proboost BSR快照（4月/5月Top100）
- 结果：37个中评候选ASIN，5大核心痛点，AI评分82/100→建议开品
- 差异化定位："Built for 3 Summers, Not 3 Months"
